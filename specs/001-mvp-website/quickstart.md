# Developer Quickstart Guide: Lovable.dev Pipeline

**Feature Branch**: `001-mvp-website`
**Created**: 2026-03-04
**Phase**: 1.3 — Developer Workflow Documentation

---

## Overview

This guide walks you through the 7-step Lovable.dev pipeline for generating website sections. The pipeline is designed for both human developers and AI agents, with a strict naming convention that makes every file self-documenting.

**Key Principle**: If you can read a filename, you know exactly what it is and where it belongs.

---

## Prerequisites

- Access to Lovable.dev (code generation platform)
- Familiarity with the project's naming convention: `{NN}-{page-name}-{section-name}`
- Access to competitor screenshots (captured and organized in `screenshots/`)
- Design guidelines file: `references/00-core/00-core-design-guidelines.md`

---

## The 7-Step Workflow

### Step 1: Identify the Section to Build

**Goal**: Determine which page and section you're building.

**Actions**:
1. Check the section-to-number mapping in `specs/001-mvp-website/research.md`
2. Identify the section number, page name, and section name

**Example**:
- Building the hero section for the landing page
- Section number: `01`
- Page name: `landing`
- Section name: `hero`
- Full identifier: `01-landing-hero`

---

### Step 2: Locate the Prompt File

**Goal**: Find the prompt file using the naming convention.

**Pattern**: `prompts/{page-name}/{NN}-{page-name}-{section-name}-prompt.md`

**Actions**:
1. Navigate to `prompts/{page-name}/`
2. Look for the file matching `{NN}-{page-name}-{section-name}-prompt.md`

**Example**:
```bash
cd prompts/landing/
ls | grep "01-landing-hero"
# Result: 01-landing-hero-prompt.md
```

**If the prompt file doesn't exist yet**:
1. Check if a prompt template exists in `specs/001-mvp-website/contracts/`
2. Copy the appropriate template (e.g., `prompt-template-hero.md`)
3. Fill in the placeholders with section-specific content
4. Append the full design guidelines from `references/00-core/00-core-design-guidelines.md`
5. Save to `prompts/{page-name}/{NN}-{page-name}-{section-name}-prompt.md`

---

### Step 3: Review Screenshot References

**Goal**: Identify which competitor screenshots to attach to the Lovable.dev prompt.

**Pattern**: `screenshots/{page-name}/{NN}-{page-name}-{section-name}/{NN}-{page-name}-{section-name}-{source}.png`

**Actions**:
1. Navigate to `screenshots/{page-name}/{NN}-{page-name}-{section-name}/`
2. List all PNG files in that directory
3. Note the filenames — these are what you'll attach in Step 4

**Example**:
```bash
cd screenshots/landing/01-landing-hero/
ls
# Result:
# 01-landing-hero-akira.png
# 01-landing-hero-datarobot.png
# 01-landing-hero-brightwave.png
```

**If screenshots are missing**:
1. Check the missing screenshots checklist in `specs/001-mvp-website/research.md`
2. Capture screenshots from competitor websites
3. Save with the correct naming pattern
4. Update the checklist

---

### Step 4: Copy Prompt + Attach Screenshots to Lovable.dev

**Goal**: Paste the prompt into Lovable.dev and attach all relevant screenshots.

**Actions**:
1. Open the prompt file from Step 2
2. Copy the entire content (including the design guidelines appendix)
3. Open Lovable.dev in your browser
4. Paste the prompt into the input field
5. Attach all screenshot files from Step 3 using the file upload button
6. Submit the prompt

**Tips**:
- Lovable.dev works best with clear, detailed prompts — don't skip the design specs section
- Attach screenshots in the order they're listed in the prompt
- If Lovable.dev asks for clarification, refer back to the design guidelines

**Example Prompt Structure**:
```markdown
# Hero Section — agentii.Solutions Landing Page

## Context
[Section description]

## Content
[Exact text content]

## Layout
[Detailed layout specs]

## Screenshots to Attach
- 01-landing-hero-akira.png — Reference headline typography
- 01-landing-hero-datarobot.png — Reference data visualization
- 01-landing-hero-brightwave.png — Reference spacing

## Design Specifications
[Typography, colors, spacing, components]

---

## Design Guidelines Appendix
[Full design guidelines pasted here]
```

---

### Step 5: Save Generated Code to Component Path

**Goal**: Save the code generated by Lovable.dev to the correct component file.

**Pattern**: `src/components/{page-subdir}/{PascalCase}.tsx` (or `src/components/{PascalCase}.tsx` for shared components)

**Actions**:
1. Copy the generated code from Lovable.dev
2. Determine the component path from the section mapping:
   - Landing page sections: `src/components/landing/{PascalCase}.tsx`
   - Finance use case sections: `src/components/usecase-finance/{PascalCase}.tsx`
   - Video use case sections: `src/components/usecase-video/{PascalCase}.tsx`
   - Shared components (Navbar, Footer): `src/components/{PascalCase}.tsx`
3. Create the file if it doesn't exist, or overwrite if regenerating
4. Save the code

**Example**:
```bash
# For 01-landing-hero:
# Component name: HeroSection
# Component path: src/components/landing/HeroSection.tsx

# Create the file
touch src/components/landing/HeroSection.tsx

# Paste the generated code and save
```

**Component Naming Convention**:
- Section name `hero` → Component name `HeroSection`
- Section name `solutions` → Component name `SolutionsSection`
- Section name `navbar` → Component name `Navbar`
- Section name `cta` → Component name `CTASection`

---

### Step 6: Refine Copy and Spacing

**Goal**: Make text edits and spacing adjustments to match the exact content and design requirements.

**Actions**:
1. Open the component file in your code editor
2. Review the generated code for:
   - Placeholder text that needs to be replaced with real content
   - Spacing values that need adjustment (check against 8px grid)
   - Color values that need correction (check against design guidelines)
   - Typography that needs adjustment (Futura for headers, Inter for body)
3. Make edits directly in the component file
4. Save changes

**Common Refinements**:
- Replace "Lorem ipsum" with actual copy
- Adjust headline text to match brand voice
- Update button labels to match CTAs
- Fix spacing to align with 8px grid (e.g., change `mt-5` to `mt-6` for 24px)
- Correct color hex values (e.g., change `bg-blue-500` to `bg-[#3B82F6]`)
- Add bilingual support (wrap text in i18n function calls)

**Example Refinement**:
```tsx
// Before (generated):
<h1 className="text-5xl font-bold">
  Your Headline Here
</h1>

// After (refined):
<h1 className="font-brand text-[56px] md:text-[72px] font-bold text-[#1F2937] leading-[1.05] tracking-[-0.02em]">
  You have the data.
  <br />
  Your agents don't.
</h1>
```

---

### Step 7: Test Responsive Layout

**Goal**: Verify the component renders correctly across desktop, tablet, and mobile breakpoints.

**Actions**:
1. Start the development server: `npm run dev`
2. Open the page in your browser
3. Test at three breakpoints:
   - **Desktop**: 1280px+ (full layout)
   - **Tablet**: 768px-1279px (2-column grids, reduced spacing)
   - **Mobile**: 375px-767px (single column, stacked buttons)
4. Check for:
   - No horizontal overflow
   - Readable text at all sizes
   - Clickable buttons (minimum 44px height)
   - Proper spacing (no elements touching)
   - Images/videos scale correctly
5. Fix any responsive issues in the component file

**Testing Tools**:
- Browser DevTools responsive mode (Cmd+Opt+M on Mac, Ctrl+Shift+M on Windows)
- Physical devices (iPhone, iPad, Android)
- Lighthouse accessibility audit (check for contrast, touch targets)

**Common Responsive Issues**:
- Text too small on mobile → Add responsive font sizes (`text-[44px] md:text-[56px] lg:text-[72px]`)
- Buttons too close together → Add gap classes (`gap-4`)
- Images overflow → Add `max-w-full` and `h-auto`
- Grid doesn't collapse → Add responsive grid classes (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`)

## Troubleshooting

### Issue: Prompt file doesn't exist

**Solution**: Generate it from a template.

```bash
# Copy the appropriate template
cp specs/001-mvp-website/contracts/prompt-template-hero.md prompts/landing/01-landing-hero-prompt.md

# Fill in the placeholders
# Append design guidelines
cat references/00-core/00-core-design-guidelines.md >> prompts/landing/01-landing-hero-prompt.md
```

### Issue: Screenshots are missing

**Solution**: Capture them from competitor websites.

1. Visit competitor website (e.g., akira.ai, datarobot.com)
2. Navigate to the relevant section (e.g., hero, solutions)
3. Take a full-page screenshot (Cmd+Shift+4 on Mac, Windows+Shift+S on Windows)
4. Save with correct naming: `{NN}-{page-name}-{section-name}-{source}.png`
5. Move to correct directory: `screenshots/{page-name}/{NN}-{page-name}-{section-name}/`

### Issue: Lovable.dev generates code that doesn't match design system

**Solution**: Check that the design guidelines were appended to the prompt.

1. Open the prompt file
2. Scroll to the bottom
3. Verify the "Design Guidelines Appendix" section exists
4. If missing, append the full content of `references/00-core/00-core-design-guidelines.md`
5. Re-submit the prompt to Lovable.dev

### Issue: Generated code has TypeScript errors

**Solution**: Run type checking and fix errors.

```bash
npm run lint
# Fix any ESLint errors

npx tsc --noEmit
# Fix any TypeScript errors
```

### Issue: Component doesn't appear on the page

**Solution**: Import and render it in the page file.

```tsx
// src/pages/Index.tsx
import HeroSection from "@/components/landing/HeroSection";

const Index = () => {
  return (
    <div className="min-h-screen bg-background">
      <Navbar />
      <HeroSection />  {/* Add this line */}
      {/* Other sections */}
    </div>
  );
};
```

## Tips for AI Agents

If you're an AI agent using this pipeline:

1. **Always follow the naming convention exactly** — no variations or shortcuts
2. **Read the prompt template first** before generating a new prompt
3. **Append the full design guidelines** to every prompt (no summaries)
4. **List all screenshot filenames** in the prompt's "Screenshots to Attach" section
5. **Verify file paths** before writing — use `ls` to check directories exist
6. **Use exact hex values** from the design guidelines, not Tailwind color names
7. **Test responsive breakpoints** by checking the generated code includes responsive classes
