# agentii.Solutions Graphic Design Guidelines

**Website:** www.agentii.net
**Brand Unit:** agentii.Solutions (Solutions Business Unit)
**Parent Brand:** agentii.ai (www.agentii.ai)
**Target Audience:** Financial Industry Professionals & Video Generation/AI Training Companies
**Design Philosophy:** Technology-Oriented Minimalism with Lattice Architecture

---

## EXECUTIVE SUMMARY

agentii.Solutions represents the **technology-forward evolution** of the agentii brand family, maintaining core brand consistency with agentii.ai while introducing enhanced technical sophistication for dual-market positioning: **financial document processing** and **video generation AI training**.

The design strategy balances:
- **Brand Continuity:** Inherits agentii.ai's bright blue signature, minimalist restraint, and lattice/table-first architecture
- **Technical Differentiation:** Introduces teal/cyan accent colors, deeper technology-oriented visual language, and enhanced data pipeline visualization
- **Dual-Audience Appeal:** Serves both conservative financial institutions and cutting-edge video AI companies through sophisticated, data-forward design

### Core Design Differentiators

**1. Enhanced Lattice Architecture with Straight Lines & Sharp Corners**
Building on agentii.ai's table-first layout, agentii.Solutions emphasizes **geometric precision** with strict 90-degree angles, grid-based layouts, and structured data flow visualizations. This reinforces both financial precision and technical pipeline architecture.

**2. Futura Typography for "agentii" Branding**
The wordmark "agentii" uses **Futura** (geometric sans-serif with strong technical heritage), while "Solutions" uses Inter/SF Pro Display for consistency with the parent brand. This creates a distinctive sub-brand identity while maintaining family cohesion.

**3. Expanded Color Palette: Blue + Teal/Cyan Technology Accents**
Retains agentii.ai's signature bright blue (#3B82F6) as primary brand color, but introduces **teal (#14B8A6)** and **cyan (#06B6D4)** as secondary technology accents to signal advanced AI/ML capabilities and differentiate from pure finance positioning.

**4. Data Pipeline & Workflow Visualization**
Inspired by DataRobot's technical aesthetic, the design prominently features **process flow diagrams, data transformation pipelines, and agent workflow visualizations** to demonstrate agentic-native data processing capabilities.

**5. Dark/Navy Hero Sections with Gradient Mesh**
Unlike agentii.ai's predominantly white background, agentii.Solutions employs **deep navy hero sections (#0F172A to #1E293B)** with subtle purple-to-teal gradient mesh overlays, creating a more technical, enterprise-software aesthetic.

## DESIGN PHILOSOPHY STATEMENT

**"Precision Through Structure, Innovation Through Data"**

agentii.Solutions communicates technical sophistication through:
- **Geometric Discipline:** Every element aligns to an 8px grid with sharp corners and straight edges
- **Data Transparency:** Live examples, interactive tables, and process visualizations replace marketing promises
- **Agentic-Native Visual Language:** Design elements reflect AI agent workflows, reasoning traces, and multimodal data processing
- **Professional Restraint:** Conservative animation and purposeful motion maintain institutional credibility

The design succeeds by **showing, not telling**—demonstrating agentic data capabilities through visual proof rather than narrative marketing.

## 3. TYPOGRAPHY SYSTEM

### 3.1 Font Stack

**Brand Wordmark Font: Futura**
- Usage: ONLY for the "agentii" wordmark and "agentii.Solutions" brand lockup
- Weights: Bold (700) for wordmark, Medium (500) for taglines adjacent to wordmark
- Fallback: `"Futura PT", "Futura", "Century Gothic", sans-serif`
- Character: Geometric, Bauhaus-inspired, technically precise — signals engineering heritage
- Note: Futura is NOT used for general headings or body copy

**Primary UI Font: Inter**
- Usage: All headings, body copy, navigation, buttons, form elements, captions
- Weights: 400 (Regular), 500 (Medium), 600 (SemiBold), 700 (Bold)
- Fallback: `"Inter", "SF Pro Display", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- Character: Highly legible geometric sans-serif, optimized for screens
- Consistency: Matches agentii.ai's primary font for brand family cohesion

**Monospace Font: JetBrains Mono (or Fira Code)**
- Usage: Code snippets, data values in tables, technical specifications, API references
- Weights: 400 (Regular), 500 (Medium)
- Fallback: `"JetBrains Mono", "Fira Code", "SF Mono", "Consolas", monospace`
- Character: Technical precision, reinforces data/engineering positioning

**Chinese Font Stack (for bilingual support):**
- Primary: `"Noto Sans SC"` (Google Fonts, free, excellent CJK coverage)
- Fallback: `"PingFang SC", "Microsoft YaHei", "Hiragino Sans GB", sans-serif`
- Note: Chinese text inherits the same size hierarchy as English but may require +1px body size for readability

### 3.3 Typography on Dark Backgrounds

On navy/dark hero sections, text colors shift:
- H1/H2: `#FFFFFF` (pure white)
- Body text: `#E2E8F0` (slate-200, softer than pure white for readability)
- Muted/secondary: `#94A3B8` (slate-400)
- Accent text: `#14B8A6` (teal) or `#3B82F6` (blue) for highlighted terms

### 3.4 Typography Rules

- Never use Futura for body copy or UI elements — it is reserved for the brand wordmark only
- Headings on light backgrounds use `#1F2937` (charcoal), never pure black `#000000`
- Maximum line length for body copy: 72 characters (approximately 680px at 16px)
- Paragraph spacing: 1.5x the body font size (24px for 16px body)
- Use `font-feature-settings: "cv01", "cv02", "ss01"` for Inter to enable distinguishable characters (1, l, I)
- Numerical data in tables should use tabular figures: `font-variant-numeric: tabular-nums`
- Financial figures must use monospace font for decimal alignment

## 4. LATTICE LAYOUT ARCHITECTURE

### 4.1 Core Grid System

The lattice architecture is the defining visual element of agentii.Solutions, directly inherited from agentii.ai and enhanced for technology-oriented presentation.

**Grid Specification:**
- Base grid: 12-column layout
- Max content width: 1280px (centered)
- Column gap: 24px (desktop), 16px (tablet), 12px (mobile)
- Outer margin: 48px (desktop), 24px (tablet), 16px (mobile)
- All elements snap to the 8px base unit grid

**Lattice Visual Characteristics:**
- Straight lines only — no curves, no rounded containers (except buttons at 4-6px radius)
- 90-degree angles at all intersections
- Visible grid lines as subtle design elements (1px, `#E5E7EB` on light, `#334155` on dark)
- Table/matrix layouts preferred over free-form card arrangements
- Horizontal and vertical rules create structured "cells" of content

### 4.2 Section Layout Patterns

- Large numbers: 48px, Inter Bold, `#3B82F6` or `#14B8A6`
- Labels: 14px, Inter Medium, `#6B7280`
- Dividers: 1px vertical lines, `#E5E7EB`
- Background: White or light gray
- Sharp corners, no rounded containers

### 4.3 Lattice Design Rules

1. **No rounded containers** for content sections — use sharp 0px radius or maximum 4px for subtle softening
2. **Buttons are the exception** — 4-6px border radius for buttons only (matching agentii.ai)
3. **Visible structure lines** — use 1px borders/dividers to reinforce the grid; the lattice should feel architectural
4. **Consistent column alignment** — text, images, and interactive elements must align to the same column edges
5. **Table-first data presentation** — when showing capabilities, metrics, or comparisons, always prefer structured tables over free-form layouts
6. **Horizontal rules between major sections** — full-width 1px dividers at `#E5E7EB` (light) or `#334155` (dark)
7. **No diagonal lines, no organic shapes** — the visual language is strictly rectilinear
8. **Card borders visible** — cards use 1px border (`#E5E7EB`) rather than shadow-only elevation, reinforcing the lattice feel

**Specifications:**
- Height: 72px (fixed)
- Background: Transparent on hero, transitions to `rgba(15, 23, 42, 0.95)` with backdrop blur on scroll
- Logo: 32px height, left-aligned with 48px left margin
- Nav links: Inter Medium 15px, `#E2E8F0` on dark, `#1F2937` on light
- Link spacing: 32px horizontal gap
- Language switcher: 14px, `#94A3B8`, right-aligned before CTA
- Primary CTA button: "CONTACT US" or "BOOK A DEMO", blue (#3B82F6), 14px uppercase, 600 weight
- Sticky behavior: Remains fixed at top with smooth transition on scroll

**Mobile Navigation (≤1023px):**
- Hamburger menu icon (right-aligned)
- Full-screen overlay menu with navy background
- Large touch targets (minimum 48px height)
- Logo remains visible in mobile header

### 5.2 Hero Section Structure

**Full-Width Dark Hero (Primary Pattern):**
- Background: Navy gradient (#0F172A to #1E293B) with radial mesh overlay
- Height: 85vh minimum (allows for scroll hint)
- Content: Centered or left-aligned within max-width container (1280px)
- H1: 56-64px, white, bold, max-width 800px
- Subheading: 18-20px, slate-200, max-width 680px, 24px margin-top
- CTA group: 32px margin-top, horizontal flex with 16px gap
- Hero visual: Pipeline diagram, data flow, or product screenshot positioned right or below text
- Scroll indicator: Subtle animated chevron or "Scroll to explore" text at bottom

**Content Hierarchy:**
1. Navigation (sticky)
2. Hero headline + subheading
3. Primary CTA + Secondary CTA
4. Hero visual (diagram/screenshot)
5. "Trusted by" logo bar (optional, at bottom of hero or top of next section)

### 5.3 Section Spacing & Rhythm

**Vertical Spacing Between Sections:**
- Large section breaks: 128px (desktop), 96px (tablet), 64px (mobile)
- Medium section breaks: 96px (desktop), 64px (tablet), 48px (mobile)
- Small section breaks: 64px (desktop), 48px (tablet), 32px (mobile)

**Within-Section Spacing:**
- Heading to body: 24px
- Body to CTA: 32px
- Between cards in grid: 24px gap (desktop), 16px (mobile)
- Between list items: 16px

**Section Background Alternation:**
- Pattern: Dark hero → White → Light gray (#F9FAFB) → Navy/Dark → White → Light gray
- Avoid two consecutive dark sections (reduces contrast fatigue)
- Use light gray for data-heavy sections (tables, comparison grids)

## 8. INTERACTIVE ELEMENTS & MICRO-INTERACTIONS

### 8.1 Animation Principles

**Conservative Motion Design:**
- Duration: 150-250ms for most transitions (fast, not sluggish)
- Easing: `cubic-bezier(0.4, 0.0, 0.2, 1)` (Material Design standard ease)
- Avoid: Bouncing, elastic, or exaggerated animations
- Purpose: Every animation must serve a functional purpose (feedback, state change, attention)

**Animation Budget:**
- Hover effects: 200ms
- Button clicks: 150ms
- Modal open/close: 250ms
- Page transitions: 300ms
- Scroll-triggered: 400ms (staggered for multiple elements)

|---------|--------|--------|---------|
| H1 | 36px | 48px | 56-64px |
| H2 | 28px | 32px | 36-42px |

### 9.3 Layout Adaptations

**Grid Columns:**
- Mobile: 1 column (stacked)
- Tablet: 2 columns
- Desktop: 3-4 columns (depending on content)

**Navigation:**
- Mobile: Hamburger menu, full-screen overlay
- Tablet: Hamburger menu or horizontal nav (depending on item count)
- Desktop: Full horizontal navigation with all items visible

**Hero Section:**
- Mobile: Stacked layout (text above visual), 70vh height
- Tablet: Stacked or side-by-side (depending on visual complexity)
- Desktop: Side-by-side (text left, visual right), 85vh height

- Mobile: Card-based layout (each row becomes a card) OR horizontal scroll
- Tablet: Horizontal scroll with sticky first column

- Mobile: 16px outer margins, 32px section spacing
- Tablet: 24px outer margins, 64px section spacing
- Desktop: 48px outer margins, 96-128px section spacing

### 9.4 Touch Targets (Mobile)

**Minimum Sizes:**
- Buttons: 44px height minimum (Apple HIG standard)
- Links in body text: 44px height touch area (even if text is smaller)
- Form inputs: 48px height minimum
- Navigation items: 48px height minimum
- Icon buttons: 44px × 44px minimum

**Spacing:**
- Minimum gap between touch targets: 8px
- Recommended gap: 12-16px

### 9.5 Mobile-Specific Optimizations

**Performance:**
- Lazy load images below the fold
- Use responsive images with `srcset` (serve smaller images to mobile)
- Defer non-critical JavaScript
- Minimize animation complexity on mobile (reduce or disable parallax, complex transforms)

**Interaction:**
- Swipe gestures for carousels/galleries
- Pull-to-refresh for data tables (if applicable)
- Bottom-sheet modals instead of centered modals (easier thumb reach)
- Sticky CTAs at bottom of screen for long pages

**Typography:**
- Increase line height slightly on mobile (1.6 → 1.7 for body text)
- Reduce letter spacing on mobile (tighter tracking for narrow screens)
- Avoid justified text (creates awkward spacing on narrow screens)

### 9.6 Tablet-Specific Considerations

**Hybrid Layouts:**
- Use 2-column layouts where desktop uses 3-4 columns
- Maintain desktop-style navigation if space permits
- Use larger touch targets than desktop (40px minimum vs 32px)

**Orientation:**
- Portrait: Closer to mobile layout (stacked, 1-2 columns)
- Landscape: Closer to desktop layout (side-by-side, 2-3 columns)

## 10. ACCESSIBILITY

### 10.1 Color Contrast Requirements

**WCAG AAA Compliance Targets:**

| Combination | Foreground | Background | Contrast Ratio | Pass Level |
|-------------|-----------|------------|----------------|------------|
| Body text on white | #1F2937 | #FFFFFF | 14.7:1 | AAA |
| Body text on off-white | #1F2937 | #F9FAFB | 14.2:1 | AAA |
| White text on navy | #FFFFFF | #0F172A | 17.1:1 | AAA |
| White text on slate | #FFFFFF | #1E293B | 13.5:1 | AAA |
| Slate-200 on navy | #E2E8F0 | #0F172A | 12.8:1 | AAA |
| Blue on white | #3B82F6 | #FFFFFF | 4.6:1 | AA (large text) |
| Teal on white | #14B8A6 | #FFFFFF | 3.4:1 | AA (large text only) |
| Teal-dark on white | #0D9488 | #FFFFFF | 4.5:1 | AA |

**Rules:**
- Body text must always meet AAA contrast (7:1 minimum)
- Large text (18px+ or 14px+ bold) must meet AA contrast (4.5:1 minimum)
- Interactive elements (buttons, links) must meet AA contrast (4.5:1 minimum)
- Teal (#14B8A6) should only be used on dark backgrounds or as large text/icons on white — for small text on white, use teal-dark (#0D9488) instead
- Never rely on color alone to convey information — always pair with text labels, icons, or patterns

### 10.2 Focus States

**Keyboard Focus Indicator:**
```css
*:focus-visible:
  outline: 2px solid #3B82F6;
  outline-offset: 2px;
  border-radius: 4px;

/* On dark backgrounds */
.dark *:focus-visible:
  outline-color: #60A5FA; /* lighter blue for visibility */
```

**Focus Order:**
- Tab order follows visual reading order (left-to-right, top-to-bottom)
- Skip-to-content link as first focusable element
- Modal focus trapping: Tab cycles within modal when open
- Focus returns to trigger element when modal closes

### 10.3 Screen Reader Support

**Semantic HTML Requirements:**
- Use `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>` landmarks
- All images require descriptive `alt` text (or `alt=""` for decorative images)
- Data tables require `<caption>`, `<thead>`, `<th scope="col|row">`
- Form inputs require associated `<label>` elements (not just placeholder text)
- Icon-only buttons require `aria-label`
- Pipeline diagrams require `aria-describedby` with text description of the workflow

**ARIA Patterns:**
- Expandable sections: `aria-expanded="true|false"`
- Tab panels: `role="tablist"`, `role="tab"`, `role="tabpanel"`
- Modal dialogs: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
- Loading states: `aria-busy="true"`, `aria-live="polite"` for status updates
- Language switcher: `lang` attribute on `<html>` and on individual elements when language differs

### 10.4 Motion & Animation Accessibility

**Reduced Motion Support:**
```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

- All animations must respect `prefers-reduced-motion`
- No auto-playing video or animation without user control
- Avoid flashing content (no more than 3 flashes per second)

### 10.5 Bilingual Accessibility

**Language Switching:**
- `<html lang="en">` for English, `<html lang="zh-Hans">` for Simplified Chinese
- Language switcher must be keyboard accessible and clearly labeled
- Screen readers must announce language change
- Chinese text may require slightly larger font sizes (+1px body) for equivalent readability

## 13. IMAGERY & VISUAL ASSETS

### 13.1 Photography Style

**If photography is used (sparingly):**
- Professional, high-quality stock or custom photography
- Cool-toned color grading (blue/teal tint)
- Subjects: Technology environments, data centers, professional settings
- Avoid: Generic stock photos of people pointing at screens, handshakes, or overly staged scenes
- Treatment: Subtle desaturation with blue/teal color overlay

### 13.2 Illustrations & Diagrams

**Primary Visual Language:**
- Line-based illustrations using the lattice aesthetic
- Straight lines, sharp corners, geometric shapes only
- Color palette: Blue, teal, cyan on white or navy backgrounds
- Stroke weight: 1.5-2px (consistent with icon system)
- No hand-drawn, organic, or playful illustration styles

**Diagram Types:**
- Data pipeline flows (horizontal, left-to-right)
- Agent workflow hierarchies (vertical, top-to-bottom)
- Before/after data transformations (side-by-side)
- Architecture diagrams (grid-based, node-and-connector)

### 13.3 Product Screenshots

**Screenshot Treatment:**
- Display in browser chrome mockup or clean device frame
- Subtle drop shadow: `0 24px 48px rgba(0, 0, 0, 0.12)`
- Border: 1px solid #E5E7EB (on light backgrounds)
- No rotation or perspective transforms — keep flat and aligned with the lattice grid
- Scale: Large enough to read key details (minimum 60% of content width)

### 13.4 Logo Usage for Client/Partner Logos

**"Trusted By" Bar:**
- All logos in grayscale (desaturated)
- Consistent height: 32-40px
- Horizontal spacing: 48-64px between logos
- Hover state: Full color on individual logo hover (optional)
- Background: White or light gray
- No borders or containers around individual logos

### 14.2 Design Consistency Checklist

When designing any page for agentii.Solutions, verify:

- [ ] Signature blue (#3B82F6) is the dominant accent color
- [ ] Teal and cyan are used as secondary accents only (not dominant)
- [ ] Lattice grid with straight lines and sharp corners is maintained
- [ ] Typography uses Inter for all UI text (Futura only for wordmark)
- [ ] 8px base unit spacing system is followed
- [ ] Buttons use 4-6px border radius (the only rounded elements)
- [ ] Data tables use visible borders (lattice style)
- [ ] Animations are conservative and purposeful
- [ ] Dark sections use navy (#0F172A) not pure black (#000000)
- [ ] All text meets WCAG AA contrast minimum (AAA preferred)
- [ ] Pipeline diagrams use straight lines only (no curves)
- [ ] Cards use 1px borders, not shadow-only elevation

## 16. CONCLUSION

agentii.Solutions' design system builds on the proven foundation of agentii.ai while introducing the technical sophistication needed to serve both financial industry professionals and video generation AI companies. The key differentiators — Futura wordmark, teal/cyan accent palette, dark hero sections, and data pipeline visualizations — create a distinct sub-brand identity without breaking family cohesion.

The lattice architecture with straight lines and sharp corners remains the defining visual signature, reinforced through every element from grid layouts to table borders to pipeline connectors. This geometric discipline communicates the precision and reliability that both target audiences demand.

Every design decision serves the core brand promise: **agentic-native data solutions that are transparent, precise, and production-ready**.
