# Overview Section — agentii.Solutions Document Processing Solution

## Context

This section provides a high-level overview of the Document Processing solution, positioned immediately below the hero. It introduces the problem (traditional document annotation fails agents) and the solution (agentic-native document intelligence). Use a white background to contrast with the dark hero above.

## Content

**Section Headline** (Futura, 36-48px, Weight 700, Color #1F2937):
Beyond OCR. Beyond Labels. Agent-Native Document Intelligence.

**Section Description** (Inter, 16-18px, Weight 400, Color #6B7280):
Traditional document processing extracts text. We extract meaning — chart semantics, table relationships, numerical precision, and the reasoning traces your agents need to make decisions autonomously.

**Three Stat Cards** (horizontal row):

Card 1:
- Metric: "50+"
- Label: "Document Formats"
- Detail: "10-K, 10-Q, bank statements, earnings reports, contracts, invoices"

Card 2:
- Metric: "98%"
- Label: "Field Accuracy"
- Detail: "Financial-grade precision with audit trails and provenance tracking"

Card 3:
- Metric: "Zero"
- Label: "Post-Processing"
- Detail: "Agent-use-ready output — no human review or data cleaning required"

**Problem/Solution Block** (2-column layout):

Left Column — "Traditional Annotation":
- Extracts text, misses chart data
- Flat labels without reasoning context
- Requires human review and cleaning
- Measures inter-annotator agreement
- Output needs post-processing for agents

Right Column — "agentii.Solutions":
- Extracts charts, tables, and numerical semantics
- Captures reasoning traces and decision paths
- Agent-validated, no human bottleneck
- Measures agent task completion rate
- Agent-use-ready from day one
