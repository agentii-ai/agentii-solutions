# agentii.Solutions Constitution

## Core Principles

### I. Agentic-Native Data Philosophy

**Data generated by agents, annotated by agents, used by agents, trained for agents.**

All data products and solutions must be designed with agentic AI systems as the primary consumer. This means:

- Data structures must be machine-readable and semantically rich for agent reasoning
- Annotation schemas must capture not just labels but reasoning traces, tool calls, and decision paths
- Output formats must be "agent-use-ready" without requiring human post-processing
- Quality metrics must measure agent task performance, not just human annotation accuracy

**Rationale**: In the agentic age, data is infrastructure for autonomous systems. Traditional human-centric annotation creates friction; agentic-native data enables seamless agent-to-agent workflows and accelerates AI capability development.

### II. Financial Industry Standards Compliance

All document processing, annotation, and data outputs MUST meet financial industry standards for:

- Numerical precision and accuracy (no rounding errors, proper decimal handling)
- Regulatory compliance (SEC filing formats, GAAP/IFRS accounting standards)
- Audit trails and provenance tracking
- Data security and privacy (SOC 2, GDPR, financial data protection)

**Rationale**: agentii.Solutions serves financial institutions where data quality directly impacts investment decisions, regulatory compliance, and fiduciary responsibilities. Sub-standard data quality is not acceptable.

### III. Multimodal-First Architecture

Solutions MUST handle rich, complex data formats natively:

- Documents with charts, tables, graphs, and numerical data (not just text extraction)
- Video with object tracking, segmentation, scene semantics, and audio tracks
- Synthetic data with reasoning traces, tool calls, and execution logs

Text-only or single-modality approaches are insufficient for modern agentic systems.

**Rationale**: Real-world business data is inherently multimodal. Financial reports combine text, tables, and charts; business meetings combine video, audio, and slides. Agents need unified multimodal understanding to perform complex tasks.

### IV. Brand Consistency with agentii.ai

The agentii.Solutions website (www.agentii.net) MUST maintain visual and brand consistency with the parent agentii.ai platform (www.agentii.ai):

- Follow the design system documented in "agentii graphic design.md"
- Use the bright blue/purple signature color (#3B82F6 or similar)
- Employ lattice/table-first layout architecture
- Maintain minimalist professional restraint
- Use geometric sans-serif typography (Inter/SF Pro Display)
- Apply data-forward design philosophy

**Rationale**: agentii.Solutions is a business unit of agentii.ai. Consistent branding builds trust, reinforces the parent brand, and creates a cohesive product family experience.

### V. Bilingual Excellence (English + Chinese)

All website content MUST be available in both English (default) and Chinese with:

- Professional translation quality (not machine-translated without review)
- Culturally appropriate messaging and examples
- Proper technical terminology in both languages
- Seamless language switching UX

**Rationale**: agentii.Solutions targets global markets including China, where data annotation and AI training are major industries. Chinese-language support is not optional—it's a market requirement.

### VI. Frontend-Only Simplicity

The website is a frontend-only static site with:

- No backend server or database
- No user authentication or session management
- Content managed through code/markdown (not CMS)
- Deployed via Vercel with GitHub as source of truth

**Rationale**: A marketing/informational website does not require backend complexity. Static site architecture ensures fast performance, easy deployment, zero server costs, and maximum reliability.

### VII. Competitive Differentiation

All content and positioning MUST clearly differentiate agentii.Solutions from competitors:

- **vs. Akira.ai (P0 competitor)**: Emphasize financial industry specialization, agentic-native data philosophy, and synthetic reasoning data
- **vs. Labellerr (P1 video competitor)**: Highlight dual-channel DiT training focus and agent-generated annotations
- **vs. Tonic.ai (P1 synthetic competitor)**: Stress reasoning data, tool-calling datasets, and agentic capability training

Avoid generic "AI data labeling" positioning—focus on the unique agentic-native value proposition.

**Rationale**: The data annotation market is crowded. Clear differentiation on agentic-native capabilities and financial industry expertise is essential for market positioning and customer acquisition.

## Brand & Design Standards

### Visual Identity

- **Primary Color**: Bright blue (#3B82F6) as signature brand color
- **Typography**: Geometric sans-serif (Inter, SF Pro Display, or similar)
- **Layout**: Lattice/table-first architecture, generous whitespace
- **Style**: Minimalist professional restraint, data-forward design
- **Motion**: Conservative animation, purposeful motion only

### Design Reference

All design decisions MUST reference and align with the design system documented in "agentii graphic design.md", including:

- Color palette (primary blue, navy/charcoal, white/off-white, light gray)
- Typography hierarchy (H1: 48-64px, H2: 32-42px, body: 16-18px)
- Spacing system (8px base unit, modular scale)
- Component design (buttons, cards, forms, dividers)
- Interactive elements and micro-interactions

### Content Presentation

- Use data visualizations and interactive tables to demonstrate capabilities
- Show concrete examples and real outputs (not just marketing promises)
- Maintain high contrast for accessibility (WCAG AAA compliance)
- Prioritize scannability with clear visual hierarchy

## Content & Localization

### Three Solution Areas

All website content MUST clearly present the three core solutions:

1. **Document Processing**: Charts, tables, numerical data extraction for financial documents (Investment, Insurance, Accounting use cases)

2. **Video & Audio Annotation**: Object tracking, segmentation, scene semantics, soundtracks for dual-channel DiT video generation model training

3. **Synthetic Data**: Reasoning data, business decision data, math/coding data with rich labels and tool callings for training reasoning LLMs and agentic capabilities

### Localization Standards

- **English**: Default language, professional business tone
- **Chinese**: Simplified Chinese (简体中文), culturally adapted messaging
- **Translation Process**: Professional review required (not raw machine translation)
- **Technical Terms**: Maintain consistency across languages
- **Examples**: Use culturally relevant examples for each market

### Messaging Hierarchy

1. **Primary Message**: "Data by agents, for agents" - agentic-native data solutions
2. **Secondary Message**: Financial industry specialization and compliance
3. **Tertiary Message**: Three solution areas with concrete use cases
4. **Supporting Message**: Competitive differentiation and technical capabilities

## Governance

### Constitution Authority

This constitution supersedes all other development practices, style guides, and technical decisions. When conflicts arise, constitution principles take precedence.

### Amendment Process

1. **Proposal**: Document proposed change with rationale and impact analysis
2. **Review**: Evaluate impact on existing features, templates, and workflows
3. **Approval**: Requires explicit approval (document in commit message)
4. **Migration**: Update all dependent templates and documentation
5. **Version Bump**: Follow semantic versioning (MAJOR.MINOR.PATCH)

### Version Semantics

- **MAJOR**: Backward-incompatible principle removals or redefinitions
- **MINOR**: New principles added or materially expanded guidance
- **PATCH**: Clarifications, wording improvements, non-semantic refinements

### Compliance Verification

All feature specifications, implementation plans, and pull requests MUST:

- Include a "Constitution Check" section verifying compliance
- Document any principle violations with explicit justification
- Demonstrate how the feature aligns with core principles
- Reference specific principles by number (e.g., "Principle III: Multimodal-First")

### Quality Gates

Before any feature can be considered complete:

- [ ] Brand consistency verified against agentii.ai design system
- [ ] Bilingual content reviewed and approved
- [ ] Agentic-native data philosophy demonstrated
- [ ] Competitive differentiation clearly articulated
- [ ] Performance requirements met (Lighthouse scores)
- [ ] Mobile responsiveness validated
- [ ] Accessibility compliance verified (WCAG AAA)

### Living Document

This constitution is a living document that evolves with the project. Regular reviews (quarterly or after major features) ensure principles remain relevant and actionable.
